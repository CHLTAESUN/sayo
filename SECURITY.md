# SAYO 보안 기준

SAYO는 보안을 기능보다 우선한다. 현재 프로젝트는 화면 검증용 프로토타입이며,
실제 회원가입과 데이터 저장을 연결하기 전에 아래 구조로 전환한다.

## 필수 구조

```text
사용자
  -> Cloudflare WAF (한국 IP만 허용)
  -> SAYO 서버 API
  -> Supabase Auth / Postgres / Storage
```

- 브라우저에서 Supabase 데이터베이스나 Storage에 직접 쓰지 않는다.
- 가입, 로그인, 게시물, 답글, 메시지, 사진 업로드는 모두 SAYO 서버 API를 통한다.
- 관리자용 서비스 키는 서버에만 저장하며 브라우저 번들에 포함하지 않는다.

## 해외 IP 차단

Cloudflare WAF Custom Rule에서 대한민국 외 요청을 차단한다.

```text
(ip.src.country ne "KR")
```

이 규칙은 중국(`CN`)을 포함한 모든 해외 국가를 차단한다. 별도 중국 차단 규칙도
방어 심도를 위해 추가할 수 있지만, 한국만 허용하는 규칙이 정상 적용되면 기능상 중복이다.

적용 대상:

- 모든 페이지
- 회원가입 및 로그인 API
- 게시물, 답글, 메시지 API
- 사진 업로드 API
- 관리자 페이지

운영자 해외 접속이 필요한 경우 국가 허용 대신 운영자 VPN의 고정 IP만 별도 허용한다.
IP 국가 판별은 완벽하지 않으며 VPN과 프록시를 완전히 차단하지 못하므로 추가 방어가 필요하다.

## 가입 및 로그인

- 이메일 또는 휴대폰 인증 필수
- 국내 본인확인기관의 PASS/휴대폰 본인인증 필수
- 본인확인기관이 제공하는 국적 또는 내·외국인 구분값 사용 가능 여부를 계약 및 법률 검토 후 결정
- 주민등록번호 원문은 수집하거나 저장하지 않음
- 중복가입 방지를 위한 CI는 암호화 또는 단방향 처리하고 접근을 엄격히 제한
- Cloudflare Turnstile 필수
- 로그인, 가입, 인증번호 요청에 IP 및 계정별 속도 제한
- 비밀번호는 직접 저장하지 않고 인증 제공자가 관리
- 관리자 계정은 피싱 방지 방식의 MFA 필수
- 반복 실패, 신규 기기, 비정상 요청은 추가 인증 또는 차단

권장 초기 제한:

| 작업 | 제한 |
| --- | --- |
| 인증번호 요청 | IP당 10분에 3회 |
| 로그인 실패 | 계정당 15분에 5회 |
| 신규 게시물 | 신규 계정은 시간당 5개 |
| 답글 | 신규 계정은 분당 3개 |
| 메시지 | 신규 계정은 분당 10개 |
| 사진 업로드 | 파일당 10MB, 이미지 형식만 허용 |

## 데이터 및 권한

- 모든 사용자 테이블에 Row Level Security 적용
- 본인 프로필 수정은 본인만 가능
- 메시지는 발신자와 수신자만 읽기 가능
- 비공개 계정 게시물은 허용된 사용자만 읽기 가능
- 차단된 사용자 간에는 조회와 메시지 전송 모두 금지
- 삭제된 콘텐츠와 계정은 정책에 따라 완전 삭제
- 운영자 작업은 감사 로그에 기록

## 사진 업로드

- 파일 확장자가 아니라 실제 MIME 형식 검사
- 이미지 재인코딩으로 숨겨진 악성 데이터 제거
- EXIF 위치 및 기기 정보 제거
- 파일 크기와 해상도 제한
- 실행 파일, SVG, HTML 업로드 금지
- 비공개 저장소와 만료되는 서명 URL 사용

## 운영

- 비밀키는 배포 환경의 Secret Manager에 저장
- 개발, 테스트, 운영 데이터베이스 분리
- 자동 백업 및 복구 훈련
- 의존성 취약점 검사
- 로그인, 권한 오류, 차단 이벤트 모니터링
- 보안 사고 대응 및 사용자 신고 절차 마련
