# 이메일 인증번호 — 나을 때 이어서 할 작업

> 코드는 이미 다 만들어서 로컬에 커밋해뒀어요. 아래 5개만 하면 인증번호 메일이 실제로 발송됩니다.
> 푸시(발행)는 일부러 안 했어요 — 4번(Supabase Confirm email OFF)을 안 하면 라이브 가입이 깨지기 때문.
> 천천히, 하나씩. 막히면 Claude한테 "이메일인증_남은작업 이어서 하자"고 하면 됩니다.

## 지금까지 한 것 (저장 완료)
- 가입을 **아이디 기반**으로 변경 (아이디@sayo.app), 이메일은 선택 입력
- **인증번호 발급/확인 버튼** 화면에 추가
- **email-verify Edge Function**(Brevo로 메일 발송) 코드 작성 → `supabase/functions/email-verify/index.ts`
- **인증번호 저장 테이블** SQL 작성 → `supabase/email_codes.sql`
- Brevo 무료 가입 완료 (Free 플랜, 카드 없음)

## 남은 작업 (전부 무료, 카드 없음)

### 1. Brevo 보내는 사람(Sender) 인증
- app.brevo.com → Settings → **Senders, Domains** → Senders
- 본인 지메일을 발신자로 등록 → 지메일로 온 확인 메일의 링크 클릭
- (가입 이메일이 이미 발신자로 자동 등록돼 있을 수도 있음 — 확인만)

### 2. Brevo API 키 발급
- app.brevo.com/settings/keys/api → **Generate a new API key**
- 이름 아무거나(sayo) → 생성 → `xkeysib-...` 키 **복사** (한 번만 보임)
- ⚠️ 채팅창에 붙여넣지 말 것. 안전한 곳에 보관.

### 3. Supabase: 테이블 만들기
- Supabase 대시보드 → SQL Editor → `supabase/email_codes.sql` 내용 붙여넣고 Run

### 4. Supabase: Confirm email 끄기 (★ 이거 안 하면 가입 안 됨)
- Authentication → Sign In / Providers → Email → **"Confirm email" OFF**
- "Allow new users to sign up" 이 **ON** 인지도 확인

### 5. Supabase: Edge Function 배포
- Edge Functions → 새 함수 **email-verify** 만들고 `supabase/functions/email-verify/index.ts` 코드 붙여넣기
- "Enforce JWT verification" **OFF**
- 함수 Secrets에 추가:
  - `BREVO_API_KEY` = (2번에서 복사한 키)
  - `BREVO_SENDER` = (1번에서 인증한 지메일 주소)

## 끝나면
- 로컬에서 `npm run dev` → 가입 화면에서 이메일 넣고 "인증번호 발급" → 지메일로 6자리 오는지 확인
- 잘 되면 `git push` (= 라이브 발행)

## 참고
- Brevo 대신 도메인이 생기면 Resend로 바꿔도 됨. 그때는 함수의 sendEmail()만 교체.
- 키(BREVO_API_KEY 등)는 Supabase Secrets에만. 코드·GitHub·vault에 저장 금지.
