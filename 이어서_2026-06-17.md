## ⏸ 열린 작업 (다음 세션 시작점)

### A. Supabase Storage 버킷 설정 — **사용자가 SQL Editor에서 1회 실행** (미실행 상태)
```sql
alter table posts add column if not exists image_url text;

insert into storage.buckets (id, name, public)
values ('post-images', 'post-images', true)
on conflict (id) do update set public = true;

create policy "post-images 공개 읽기"
  on storage.objects for select using (bucket_id = 'post-images');

create policy "post-images 본인 업로드"
  on storage.objects for insert to authenticated
  with check (bucket_id = 'post-images' and (storage.foldername(name))[1] = auth.uid()::text);
```
- 코드 경로가 `post-images/{uid}/...` 라서 정책이 폴더 첫칸=본인 uid만 허용(보안).
- `policy already exists` 나오면 그 줄 빼고 나머지만 실행.
- 실행 후: 라이브서 로그인→사진 올리기→새로고침해도 사진 남으면 완료. (현재 image_url 컬럼 존재 여부 미확인 — 위 alter가 처리)
